# Winston AI MCP Server

MCP server wrapper for [Winston AI](https://github.com/winston-ai/winston-ai-mcp) — upstream package `winston-ai-mcp`.

## What is Winston AI?

MCP server for [Winston AI](https://gowinston.ai/) — AI content detection platform. Analyze text to determine if it was written by a human or generated by AI. Useful for content moderation, academic integrity, and editorial workflows.

**API key required** — sign up at [gowinston.ai](https://gowinston.ai/).

**Summary.** Winston AI MCP Server — Dockerized from upstream `winston-ai-mcp` package.

## Example Prompts

Here are example prompts you can use with Claude (or any MCP client) when this tool is connected:

- "Check if this text was written by AI."
- "Analyze this article for AI-generated content."

## Deploy

### Docker Compose (recommended)

```bash
docker-compose up -d
```

### Docker Run (stdio mode)

```bash
docker run -i --rm \
  -e WINSTONAI_API_KEY \
  hackerdogs/winston-ai-mcp:latest
```

### Docker Run (HTTP streamable mode)

```bash
docker run -d -p 8674:8674 \
  -e MCP_TRANSPORT=streamable-http \
  -e MCP_PORT=8674 \
  -e WINSTONAI_API_KEY \
  hackerdogs/winston-ai-mcp:latest
```

## MCP Client Configuration

### Stdio mode (default)

Add to your Claude Desktop or Cursor MCP config:

```json
{
  "mcpServers": {
    "winston-ai-mcp": {
      "command": "docker",
      "args": [
        "run",
        "-i",
        "--rm",
        "-e",
        "MCP_TRANSPORT",
        "-e",
        "WINSTONAI_API_KEY",
        "hackerdogs/winston-ai-mcp:latest"
      ],
      "env": {
        "MCP_TRANSPORT": "stdio",
        "WINSTONAI_API_KEY": ""
      }
    }
  }
}
```

### HTTP mode (streamable-http)

First, start the server using Docker Compose or `docker run` with HTTP mode (see [Deploy](#deploy) above), then point your MCP client at the running server:

```json
{
  "mcpServers": {
    "winston-ai-mcp": {
      "url": "http://localhost:8674/mcp"
    }
  }
}
```

> **When to use HTTP mode:** HTTP mode is ideal for shared/remote deployments, multi-user setups, and [Hackerdogs](https://hackerdogs.ai) scheduled prompts. The server runs as a long-lived process and accepts connections from multiple MCP clients concurrently.

## Environment Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `MCP_TRANSPORT` | `stdio` | Transport mode: `stdio` or `streamable-http` |
| `MCP_PORT` | `8674` | HTTP port (only used with `streamable-http`) |
| `WINSTONAI_API_KEY` | — | Winston AI API key — get one at [gowinston.ai](https://gowinston.ai/) |

## Testing

### Automated tests

```bash
./test.sh
```

### Test directly with Docker

**1. Start the server in HTTP mode:**

```bash
docker run -d --rm --name winston-ai-mcp-test -p 8674:8674 \
  -e MCP_TRANSPORT=streamable-http \
  -e WINSTONAI_API_KEY \
  hackerdogs/winston-ai-mcp:latest
```

**2. Initialize the MCP session:**

```bash
SESSION_ID=$(curl -s -D - -X POST http://localhost:8674/mcp \
  -H "Content-Type: application/json" \
  -H "Accept: application/json, text/event-stream" \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2024-11-05","capabilities":{},"clientInfo":{"name":"test","version":"0.1"}}}' \
  2>&1 | grep -i mcp-session-id | awk '{print $2}' | tr -d '\r\n')

curl -s -X POST http://localhost:8674/mcp \
  -H "Content-Type: application/json" \
  -H "Accept: application/json, text/event-stream" \
  -H "mcp-session-id: $SESSION_ID" \
  -d '{"jsonrpc":"2.0","method":"notifications/initialized"}'
```

**3. List available tools:**

```bash
curl -s -X POST http://localhost:8674/mcp \
  -H "Content-Type: application/json" \
  -H "Accept: application/json, text/event-stream" \
  -H "mcp-session-id: $SESSION_ID" \
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/list","params":{}}'
```

**4. Clean up:**

```bash
docker stop winston-ai-mcp-test
```
